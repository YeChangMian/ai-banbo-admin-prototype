# AI伴播管理后台 PRD（简化版 V2）

## 1. 文档信息

- 产品名称：AI伴播
- 版本：V2 简化原型
- 更新时间：2026-07-17
- 原版本存档：`archive/v1/`

## 2. 产品定位

AI伴播面向服饰、美妆、宠物等内容电商行业中，具备固定直播频次和一定商品规模的品牌商家、抖店运营团队及直播代运营团队。

核心用户包括直播运营、中控、内容制作人员和团队管理员。这些用户熟悉商品和直播业务，但通常不理解动作流、生成子任务、模型参数等 AI 生产概念。

产品解决三个具体问题：

1. 主播无法在有限时间内展示全部商品款色和使用效果。
2. 直播团队制作伴播视频依赖专业人员，准备周期长、审核成本高。
3. 直播过程中缺少可按商品讲解、主播口令、评论或活动节点触发的辅助内容。

## 3. 版本目标

V2 的目标不是减少功能数量，而是减少用户必须理解的概念和决策。

普通用户只需要理解四类对象：

- 商品
- 数字模特
- 展示模版
- 伴播视频

主要制作流程收敛为：

`选择商品 → 选择数字模特 → 选择展示模版 → 生成并审核`

动作、动作流、生成任务、子任务和技术状态仍可作为系统内部能力存在，但不在普通用户主流程中直接展示。

## 4. 信息架构

### 4.1 工作台

展示当天最需要关注的信息：

- 正在直播：直播账号、当前视频封面、视频名称、伴播时长、播放次数。
- 今日数据：伴播时长、播放视频数、生成视频数、待审核数。
- 待处理：待审核视频、需要重新生成的视频、新同步商品。

工作台不展示商品、项目结构或系统能力说明，避免重复信息。

### 4.2 伴播内容

子菜单：

- 制作伴播
- 伴播视频

### 4.3 素材库

子菜单：

- 商品
- 数字模特
- 展示模版

### 4.4 设置

子菜单：

- 账号与店铺
- 团队管理

父菜单支持展开和收起。移动端通过菜单按钮打开侧边导航。

## 5. 核心功能

### 5.1 制作伴播

#### 步骤一：选择商品

- 支持按商品名称或商品 ID 搜索。
- 支持多选商品。
- 商品卡片展示主图、名称、类目和可用款色。
- 默认按 SPU 选择；只有需要指定款色时才进一步展开 SKU。

#### 步骤二：选择数字模特

- 一次制作仅选择一个数字模特。
- 卡片展示标准半身白底图、名称和身形特征。
- 系统自动调用该模特的标准脸部、表情和身体资产。

#### 步骤三：选择展示模版

- 用户选择想要的展示效果，不需要自行组合动作。
- 模版展示封面、名称、动作顺序、适用部位和款色能力。
- 例如：正面展示、细节展示、多款色轮播、活动互动。

#### 步骤四：生成与审核

- 开始生成后展示整体进度。
- 系统自动完成商品款色识别、动作匹配、视频生成和基础质检。
- 完成后按商品展示审核结果。
- 支持预览、单条通过、重新生成和全部通过。
- 审核通过的视频自动进入伴播视频库。

### 5.2 伴播视频

- 统一展示审核通过的生成视频和直接上传的视频。
- 支持按名称或关联商品搜索。
- 列表展示封面、名称、关联商品、触发方式、播放次数、创建人。
- 支持播放和设置触发方式。
- 触发方式在最终视频层配置，避免在动作、动作流和视频中重复设置。

### 5.3 商品

- 商品由授权店铺同步，也可保留人工导入能力。
- 支持按商品名称或商品 ID 搜索。
- 展示商品图片、名称、商品 ID、二级类目、近七日销量、伴播视频数和抖店状态。
- 商品分析识别模特图、不同款色白底图和细节图。
- 颜色属于 SKU 信息，不作为普通商品标签重复展示。

### 5.4 数字模特

- 卡片默认展示抠图后的正面半身白底图。
- 数字模特资产包括人脸素材、表情素材、全身素材和身体结构信息。
- 普通制作用户只选择模特，不需要查看拆解过程。
- 资产管理员可进入详情查看和维护标准素材。

### 5.5 展示模版

- 展示模版是经过配置的完整展示效果，供制作伴播时直接选择。
- 模版内部可包含一个或多个动作。
- 展示适用部位、单款色或多款色能力、动作顺序和使用次数。
- 复杂的动作编辑能力只向管理员开放。

### 5.6 账号与店铺

- 集中管理直播账号接入和抖店授权。
- 直播账号用于直播间连接和伴播控制。
- 店铺用于同步商品。
- 两类账号在同一页面分区展示，减少设置菜单层级。

### 5.7 团队管理

- 角色层级：系统管理员 > 管理员 > 团队成员。
- 上级可管理下级，同级和下级不可管理上级。
- 系统中必须存在至少一个系统管理员。
- 支持新增成员、修改成员信息、关闭权限和恢复权限。
- 所有会改变账号或权限的操作均需二次确认。

## 6. 简化规则

1. 一个字段只在对用户决策有帮助时展示。
2. 同一信息不在菜单、列表和详情中重复说明。
3. 默认展示常用选择，高级配置按需展开。
4. 系统状态转译为用户结果，例如“生成中”“待审核”，不展示内部子任务。
5. 批量操作只在用户勾选内容后出现。
6. 可直接播放、停止、调整队列的直播操作不增加二次确认。
7. 删除、权限变更、解除授权等不可逆或高风险操作必须二次确认。

## 7. 权限策略

- 团队成员：制作伴播、审核自己创建的视频、使用伴播视频和查看素材。
- 管理员：拥有团队成员能力，可维护展示模版并管理团队成员。
- 系统管理员：拥有全部能力，可管理管理员、账号接入和店铺授权。

不可操作的数据应直接禁用勾选和操作入口，不在用户点击后再提示无权限。

## 8. 原型范围

本版本为静态交互原型，覆盖：

- 后台信息架构和响应式布局
- 伴播制作四步流程
- 模拟生成进度和审核交互
- 商品、数字模特、展示模版和伴播视频管理
- 账号店铺和团队管理弹窗
- V1 复杂版存档入口

暂不包含：

- 抖音开放平台真实授权和数据同步
- AI 视频生成接口
- 数据持久化
- 消息通知和计费结算
- 浏览器插件 V2 改造

## 9. 验收标准

1. 新用户进入系统后，可在不理解动作流和生成任务的情况下完成一批伴播视频制作。
2. 主流程不超过四个步骤，每一步只有一个主要决策。
3. 所有父菜单支持展开和收起。
4. 桌面端和移动端无横向溢出、文字遮挡和控件重叠。
5. 原复杂版本可通过 `archive/v1/` 独立访问。
6. 生成结果必须经过人工审核后进入伴播视频库。
