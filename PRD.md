# AI伴播管理后台 PRD（流程简化版 V2.1）

## 1. 文档信息

- 产品名称：AI伴播
- 版本：V2.1 流程简化原型
- 更新时间：2026-07-17
- 原版本存档：`archive/v1/`
- 字段精简试验版存档：`archive/v2-minimal/`

## 2. 产品定位

AI伴播面向服饰、美妆、宠物等内容电商行业中，具备固定直播频次和一定商品规模的品牌商家、抖店运营团队及直播代运营团队。

核心用户包括直播运营、中控、内容制作人员和团队管理员。这些用户熟悉商品和直播业务，但通常不理解动作流、生成子任务、模型参数等 AI 生产概念。

产品解决三个具体问题：

1. 主播无法在有限时间内展示全部商品款色和使用效果。
2. 直播团队制作伴播视频依赖专业人员，准备周期长、审核成本高。
3. 直播过程中缺少可按商品讲解、主播口令、评论或活动节点触发的辅助内容。

## 3. 版本目标

V2.1 的目标是减少用户必须理解的概念和操作步骤，不减少业务判断所需的信息。

本版本必须区分：

- 功能简化：减少菜单层级、重复入口、重复配置和内部任务概念。
- 信息完整：列表、详情、结果确认和管理页面继续展示用户判断与操作所需的完整字段。

不得以“简化”为理由删除商品图片类型、SKU 款色、触发条件、创建信息、账号状态等有效业务字段。

普通用户只需要理解四类对象：

- 商品列表
- 数字模特
- 展示模版
- 伴播视频

主要制作流程收敛为：

`选择商品 → 选择数字模特 → 选择展示模版 → 制作并确认`

动作、动作流、生成任务、子任务和技术状态仍可作为系统内部能力存在，但不在普通用户主流程中直接展示。

## 4. 信息架构

### 4.1 工作台

展示当天最需要关注的信息：

- 正在直播：直播账号、当前视频封面、视频名称、伴播时长、播放次数。
- 数据统计支持日、周、月和自定义时间。
- 伴播数据：伴播时长、触发次数、伴播商品数、播放视频数和趋势图。
- 生成数据：生成视频数量、生成时长、消耗金额、重新生成率和趋势图。
- 待处理：待确认视频、需要重新生成的视频、新同步商品。

工作台不展示商品、项目结构或系统能力说明，避免重复信息。

### 4.2 伴播内容

子菜单：

- 制作伴播
- 伴播视频

### 4.3 素材库

子菜单：

- 商品列表
- 数字模特
- 展示模版

### 4.4 设置

子菜单：

- 账号与店铺
- 团队管理

父菜单支持展开和收起。移动端通过菜单按钮打开侧边导航。

## 5. 核心功能

### 5.1 制作伴播

#### 步骤一：选择商品

- 支持按商品名称或商品 ID 搜索。
- 商品为选填项，支持多选；不选择商品时，可继续制作不关联商品的伴播视频。
- 桌面端商品选择区每行展示 5 个商品卡片，窄屏根据可用宽度自动减少列数。
- 商品卡片展示主图、名称、类目、可用款色和已有伴播视频数量；类目与款色合并为普通文本“类目｜款色”，不使用标签样式；点击数量跳转“伴播视频”并按商品 ID 搜索。
- 默认按 SPU 选择；只有需要指定款色时才进一步展开 SKU。

#### 步骤二：选择数字模特

- 一次制作仅选择一个数字模特。
- 选择卡片展示标准半身白底图、名称和身形特征。
- 系统自动使用该模特已解析的人脸、表情和身形信息。

#### 步骤三：选择展示模版

- 用户选择想要的展示效果，不需要自行组合动作。
- 模版展示封面、名称、动作顺序、适用部位和款色能力。
- 例如：正面展示、细节展示、多款色轮播、活动互动。

#### 步骤四：制作与确认

- 第三步选择展示模版后，主按钮由“下一步”改为“制作视频”。
- 点击“制作视频”进入第四步并立即展示整体生成进度，不再要求用户二次点击开始生成。
- 进度条同时展示百分比和视频数量进度，例如“5 / 16”。
- 制作过程中原“上一步”改为“停止任务”，不可返回修改前序选择。
- 停止任务后立即展示已生成视频供用户确认，并汇总“已生成视频、未生成、生成失败”数量；数量为 0 的统计项不展示。
- 停止任务后只为已生成视频创建确认卡片，未生成和失败视频不进入待确认列表。
- 系统自动完成商品款色识别、动作匹配、视频生成和基础质检。
- 完成后按商品展示待确认结果。
- 支持预览、单条确认、重新生成和全部确认，不使用“审核”“通过”等流程化文案。
- 生成结果的预览弹窗底部提供“确认”按钮，确认后同步更新对应结果卡片；其他图片或视频预览不展示该按钮。
- 生成结果的预览弹窗同时提供“重新生成”和“弃用”：重新生成仅更新当前视频，弃用需二次确认，弃用后移出本次确认且不进入“伴播视频”。
- 当本次结果包含弃用视频时，处理完成页展示“本次结果已处理完成”，避免误写为“已全部确认”。
- 单条确认后，该结果立即从当前确认页移除并进入“伴播视频”列表；全部确认时批量移入伴播视频列表。
- 生成结果在桌面端每行展示 3 个卡片，窄屏自动降为单列。
- 全部视频确认完成后，第四步展示“已全部确认”和“继续制作”按钮，不自动跳转。
- 用户点击“继续制作”后，系统清空本次选择并返回第一步。
- 确认后的视频自动进入伴播视频库。

### 5.2 伴播视频

- 视频类型仅包括“生成视频”和“上传视频”。
- 生成视频由商品和展示模版生成，名称规则为“货号_展示模版名称”；货号随抖店商品同步进入系统。
- 上传视频不单独填写名称，视频名称直接使用上传文件名。
- 支持按名称或关联商品搜索。
- 列表展示选择框、视频类型、视频封面、视频 ID、名称、展示模版、关联商品白底图、货号、商品名称、商品 ID、触发方式、播放次数、创建人和创建时间。
- 支持类型、触发方式、创建人筛选。
- 支持批量修改触发条件和批量删除。
- 伴播视频列表不设置单独操作列，不提供“播放”“查看审核”或“详情”文字操作。
- 点击任意视频封面直接弹窗播放视频；商品图片仍保持图片放大预览交互。
- 触发方式在最终视频层配置，避免在动作、动作流和视频中重复设置。

### 5.3 商品列表

- 商品由授权店铺同步，也可保留人工导入能力。
- 支持按商品名称或商品 ID 搜索。
- 顶部展示当前店铺、切换店铺、最近同步时间和同步商品操作。
- 列表展示勾选、模特图、不同款色白底图、商品名称、商品 ID、二级类目、标签、近七日销量、抖店状态和伴播视频数量，数量为 0 时仍展示“0 条”。
- 点击伴播视频数量，跳转“伴播视频”菜单并自动按当前商品 ID 搜索。
- 商品分析识别模特图和不同款色白底图；每张白底图必须标注对应款色。
- 类目、标签和抖店商品状态在表头筛选，标签和类目支持搜索选择。
- 支持全选、批量进入制作伴播、分页和每页 20/50 条切换。
- 颜色属于 SKU 信息，不作为普通商品标签重复展示。

### 5.4 数字模特

- 数字模特分为“真人形象”和“动画形象”两类，但两类资产在同一页面混合展示，不使用 Tab 或分类页面。
- 卡片图片右上角标识形象类型；搜索和数量统计覆盖页面内全部数字模特。
- 创建数字模特时需要选择形象类型、模特名称、1 张正脸图片；参考图片和形象视频为选填素材。
- 数字模特页面只支持按模特名称或标签搜索，不提供体型筛选。
- 系统根据上传素材自动生成抠图后的正面半身白底图，卡片左下角展示正面脸部白底图。
- 卡片文字只展示模特名称和创建时间，避免卡片信息干扰图片判断。
- 数字模特资产包括人脸素材、表情素材、身形素材和身体结构信息。
- 创建弹窗支持“1 张正脸图片 + 多张参考图片 + 选填形象视频”。正脸图片用于锁定核心识别特征；参考图片最多 9 张，用于补充全身正面、侧面、背面、表情、典型动作和道具互动；形象视频用于补充连续动作、转身和动态形变。
- 真人形象重点解析人脸、表情、身形和身体比例；动画形象重点解析固定角色特征、画风、头身比例、服饰配色、纹理和动态特征。
- 动画数字模特详情的参考图片区必须首先展示正脸图片，再展示全身、动作和道具等辅助参考图片。
- 详情展示正脸、45 度侧脸、90 度侧脸；微笑、讲话、开心、低头、抬头；全身正面、侧面、背面。
- 详情按“模特概览、人脸与表情、身形与身体信息”分区展示，不重复展示创建时上传的原始素材；身体信息包括头身比、头肩比、肩腰臀比例、体型标签、腰线位置和臀线位置。

### 5.5 展示模版（动作）

- 展示模版菜单只展示单个动作，不展示动作流。
- 数据块展示触发方式、封面、模版名称、动作描述、“单款色/多款色｜展示类目”、使用次数、创建人和创建时间，不展示标签。
- 详情完整展示模版信息、生成方式、参考素材、动作提示词、音频、展示能力、触发方式和触发配置。
- 新增动作沿用三步创建流程：基础信息 → 生成方式 → 触发方式。
- 基础信息配置动作名称和类型。
- 生成方式支持“参考图 + 提示词”和“参考视频”，并配置展示类目、展示款色及选填音频。
- 触发方式支持商品讲解、主播口令、评论和活动节点多选；对应触发类型支持配置关键词或活动节点。
- 步骤按钮随流程变化：第一步仅展示“下一步”，第二步展示“上一步、下一步”，第三步展示“上一步、开始分析”。
- 进入下一步前校验当前步骤必填项。参考图方式必须上传参考图并填写提示词；参考视频方式必须上传参考视频；主播口令和评论触发必须填写对应关键词。
- 点击“开始分析”后在当前弹窗展示分析进度，依次进行素材完整性检查、动作视频生成、动作姿态与镜头节奏识别和封面生成。
- 创建弹窗高度不超过当前浏览器视口，头部步骤和底部操作区固定，仅中间内容区滚动；复选框和单选框使用系统统一尺寸。
- 分析过程包括素材检查、动作视频生成、动作姿态与镜头节奏识别和封面提取；分析进行中不可误关闭弹窗。
- 分析完成后展示动作视频预览、封面、展示范围和触发方式摘要，并支持返回修改。用户点击“确认创建”后，展示模版才进入模版列表。
- 制作伴播时由系统根据动作能力自动组织生成方案，用户不需要配置动作流。

### 5.6 账号与店铺

- 集中管理直播账号接入和抖店授权。
- 直播账号用于直播间连接和伴播控制。
- 店铺用于同步商品。
- 两类账号在同一页面分区展示，减少设置菜单层级，但各自字段保持完整。
- 直播账号展示头像、名称、抖音号、接入时间、接入人和移除操作。
- 店铺展示店铺头像、名称、店铺 ID、授权状态、授权到期时间和对应操作。

### 5.7 团队管理

- 角色层级：系统管理员 > 管理员 > 团队成员。
- 上级可管理下级，同级和下级不可管理上级。
- 系统中必须存在至少一个系统管理员。
- 支持新增成员、修改成员信息、关闭权限和恢复权限。
- 列表展示账号名称、手机号、权限角色、账号状态和操作。
- 所有会改变账号或权限的操作均需二次确认。

### 5.8 直播中控台浏览器插件

- 伴播控制顶部提供“数字模特”选择，默认选项展示为“全部”，也可指定一个数字模特；下方说明为“仅自动触发所选模特的伴播视频，其他模特视频需手动选择播放”。
- 指定数字模特后，商品讲解等自动触发仅从该模特的视频中选择；当前正在播放的视频不被强制中断。
- 手动“立即播放”和“下一条播放”不受数字模特筛选限制，允许人工选择其他模特的视频。
- 当前播放、播放队列、商品视频浮窗和伴播视频列表均展示数字模特名称，帮助中控判断将要播放的形象。
- 插件端直播商品列表中，有伴播视频的商品统一展示“1 个伴播视频”，悬浮面板同步展示 1 条可手动播放的视频；无视频商品不展示入口。

## 6. 简化规则

1. 不删除对用户判断、筛选、追溯或操作有帮助的字段。
2. 同一信息不在菜单、列表和详情中重复说明。
3. 完整信息可以通过列表、详情和按需展开分层展示，但不能直接丢失。
4. 所有详情信息每行只展示一个字段，字段名固定宽度、字段值左对齐，长内容自然换行。
5. 系统状态转译为用户结果，例如“制作中”“待确认”，不展示内部子任务。
6. 批量操作只在用户勾选内容后出现。
7. 可直接播放、停止、调整队列的直播操作不增加二次确认。
8. 删除、权限变更、解除授权等不可逆或高风险操作必须二次确认。

## 7. 权限策略

- 团队成员：制作伴播、确认自己创建的视频、使用伴播视频和查看素材。
- 管理员：拥有团队成员能力，可维护展示模版并管理团队成员。
- 系统管理员：拥有全部能力，可管理管理员、账号接入和店铺授权。

不可操作的数据应直接禁用勾选和操作入口，不在用户点击后再提示无权限。

## 8. 原型范围

本版本为静态交互原型，覆盖：

- 后台信息架构和响应式布局
- 伴播制作四步流程
- 模拟生成进度和结果确认交互
- 商品、数字模特、展示模版和伴播视频管理
- 账号店铺和团队管理弹窗
- 直播中控台浏览器插件及数字模特自动触发控制
- V1 复杂版存档入口

暂不包含：

- 抖音开放平台真实授权和数据同步
- AI 视频生成接口
- 数据持久化
- 消息通知和计费结算

## 9. 验收标准

1. 新用户进入系统后，可在不理解动作流和生成任务的情况下完成一批伴播视频制作。
2. 主流程不超过四个步骤，每一步只有一个主要决策。
3. 所有父菜单支持展开和收起。
4. 桌面端和移动端页面本身无横向溢出、文字遮挡和控件重叠；字段较多的表格在表格容器内横向滚动。
5. 原复杂版本可通过 `archive/v1/` 独立访问。
6. 生成结果必须经过人工确认后进入伴播视频库；全部确认后展示完成状态，点击“继续制作”才返回第一步。
7. 商品、伴播视频、数字模特、展示模版、账号与团队页面具备本 PRD 要求的完整展示字段。
